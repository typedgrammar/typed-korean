# Parsing-accuracy eval

Keeps the project's Japanese-parsing quality **quantifiable** and **trending up**.
Two complementary evals, both surfaced live in the playground's **Eval** tab.

## 1. Hard codex eval (primary)

Runs the **real `codex exec` annotate pipeline** end-to-end on a set of sentences
(`eval-cases.json`, medium → very-hard) and measures how well codex turns each
sentence into a Typed Japanese snippet that **type-checks AND resolves
byte-identically** to the target.

Two signals (don't conflate them):
- **reliability** = passes / samples per case → *codex accuracy* (the headline KPI).
- **capability** = did codex ever resolve it → *can the library express it at all?*
  A case with capability 0 is a candidate **library gap** → `gap-backlog.md`.
- **modelable** = capability excluding classical-literal cases (which are
  unmodelable by design and kept as `SuffixPart` literals).

### Run a round (on-demand)

```
# 1. run the real pipeline (K samples/case; each sample = one real codex call)
node scripts/eval-codex.mjs --round N --samples 1 --concurrency 4 > /tmp/r.json

# 2. record: history snapshot + scoreboard + playground presets + (gap classify)
node scripts/eval-codex-record.mjs /tmp/r.json

# 3. triage failures → apply ONLY clean, gated fixes:
node scripts/eval-gate.mjs        # snippets + vocab + typecheck + lib + annotate
#   ts-def gaps (src/*.d.ts, additive) preferred; prompt fixes (annotate.ts) 2nd;
#   classical = leave. Keep iff gate stays green, then re-run the case to confirm.
```

It is **on-demand** — there is no always-on timer. Trigger a round when you have a
hypothesis to test (after a fix) or to refresh the published numbers.

### Files
- `eval-cases.json` — the case set (id, jp, reading, en, zh, category, difficulty, modelability).
- `scripts/eval-codex.mjs` — runner (real codex, N-sample, auto-diff drift).
- `scripts/eval-codex-record.mjs` — records history + `hard-scoreboard.md` + presets.
- `hard-scoreboard.md` — per-round reliability / capability / by-difficulty.
- `gap-backlog.md` — discovered ts-def gaps (fixed / open / won't-fix).
- `history/hard-NNN.json` — full per-round result incl. each case's codex snippet + drift.
- New hard cases are generated by the `hard-eval-cases` / `medium-eval-cases` workflows.

## 2. Fixed rubric benchmark (secondary / legacy)

A frozen 12-item benchmark scored by independent judges against
[`rubric.md`](./rubric.md) (D1–D6, 0/1/2). Comparable across rounds; reached its
ceiling (97.5%). Kept for regression signal on the curated corpus.

```
node scripts/eval-bench.mjs --round N > /tmp/b.json   # deterministic batch
#  → cross-review via the review.workflow.js Workflow (args: round, rubric, items)
node scripts/eval-record.mjs <result.json> benchmark-scoreboard.md
```

- `benchmark.json` / `benchmark-scoreboard.md` — the fixed set + its KPI table.
- `review.workflow.js` — the judges-+-synthesizer cross-review.
- `history/bench-NNN.json`, `findings/bench-NNN.md` — per-round detail.

## Fix routing (both evals)

Every fix is **ts-def** (library type gap → `src/*.d.ts`, durable, preferred),
**prompt** (`scripts/annotate.ts` hard-rules), or **content** (one-off). Every
applied fix must keep `eval-gate.mjs` green (so an eval win that breaks the 384-
snippet course corpus is rejected) — otherwise revert and log to the backlog.
